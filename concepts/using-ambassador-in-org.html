Using Ambassador in your organization

Ambassador can be used in a variety of situations. We've documented a few common use cases with Ambassador:

Transitioning from a monolith to microservices

The "strangler" pattern is a common approach to incrementally migrating from a monolith to microservices. Ambassador can be deployed as a facade to both the monolith and microservices, enabling a migration to Kubernetes and microservices without impacting end users. Read more here.

Testing

Ambassador can be used for testing services and accelerating continuous delivery. With features such as canary releases and traffic shadowing, Ambassador enables rapid iteration of services. Read more here.
